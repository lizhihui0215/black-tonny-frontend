# Dashboard 交互与联动规则

## 1. 日期选择器

Dashboard 统一支持以下预设：

- `today`
- `yesterday`
- `last7days`
- `last30days`
- `thisMonth`
- `lastMonth`
- `custom`

## 2. 页面统一筛选状态

页面所有模块共用统一筛选状态，建议包含：

- `preset`
- `startDate`
- `endDate`
- `compareStartDate`
- `compareEndDate`

页面所有数据请求基于同一份筛选状态执行，避免模块间时间口径不一致。

## 3. 对比规则

全页面统一使用：

**当前区间 vs 上一个等长区间**

### 示例

- 今天 → 对比昨天
- 近 7 天 → 对比前 7 天
- 近 30 天 → 对比前 30 天
- 本月 → 对比上一个等长区间

v1 不建议不同模块使用不同对比逻辑。

## 4. 顶部 8 张卡联动规则

### 4.1 时间累计型指标

以下指标主值完全跟随日期变化：

- 销售额 `salesAmount`
- 订单数 `orderCount`
- 客单价 `avgOrderValue`
- 销售件数 `salesQuantity`
- 连带率 `attachRate`

### 说明

用户切换：

- 今天
- 近 7 天
- 本月

这些卡片主值都需要刷新。

### 4.2 当前状态型指标

以下指标主值恒为当前值，不随日期主值变化：

- 低库存 SKU 数 `lowStockSkuCount`
- 缺码款数 `sizeBreakStyleCount`
- 过季库存件数 `outOfSeasonStockQty`

### 说明

这类指标本质是“当前风险状态”，不是“时间区间累计值”。

如果主值也跟日期切换，会导致用户误解。

### 4.3 当前状态型指标的副值联动

虽然主值恒为当前值，但副值仍需跟所选周期联动。

### 示例

- 低库存 SKU 数：近 7 天新增 3 个
- 缺码款数：近 7 天新增 2 款
- 过季库存件数：较上期减少 18 件

## 5. 卡片点击跳转规则

- 销售额 → 销售明细页
- 订单数 → 订单列表页
- 客单价 → 销售分析页
- 销售件数 → 商品销售分析页
- 连带率 → 商品组合分析页
- 低库存 SKU 数 → 库存预警页
- 缺码款数 → 缺码明细页
- 过季库存件数 → 过季库存页

## 6. 模块联动建议

### 6.1 销售趋势模块

跟随日期变化，承接以下指标：

- 销售额
- 订单数
- 销售件数

### 6.2 热销商品 / 热销尺码模块

跟随日期变化，承接以下指标：

- 客单价
- 销售件数
- 连带率

### 6.3 库存风险模块

主值以当前状态为主，承接以下指标：

- 低库存 SKU 数
- 缺码款数
- 过季库存件数

### 6.4 会员模块

建议跟随日期变化，但不进入顶部 8 卡。

## 7. 展示规则

### 7.1 金额格式

- 销售额、客单价显示金额格式
- 例如：`￥12,860`

### 7.2 比例与变化

- 销售额、订单数、客单价、销售件数：显示百分比变化
- 连带率：显示绝对值变化，例如 `+0.2`
- 库存状态型指标：显示绝对数量变化，例如 `+3`

### 7.3 单位显示

- 单位由接口返回
- 前端不自行推断

## 8. 空态与异常态规则

### 8.1 无销售数据

当某时间区间没有销售数据时：

- 销售额 = 0
- 订单数 = 0
- 客单价 = 0
- 销售件数 = 0
- 连带率 = 0

### 8.2 对比周期为 0

当对比周期值为 0 时：

- `compareValue = null`
- 前端显示 `--`

### 8.3 当前状态型指标无变化

当无变化时：

- `compareDirection = flat`
- `compareValue = 0`
- 前端显示 `较上期无变化`

## 9. 前后端职责边界

### 后端负责

- 指标计算
- 日期区间计算
- compare 值计算
- compareDirection 计算
- 单位与副文案返回

### 前端负责

- 日期控件维护
- 调用接口
- 渲染 8 张卡
- 根据 `compareDirection` 渲染颜色和图标
- 根据 `compareType` 渲染百分比或绝对值

## 10. 实现原则

1. 所有模块使用统一筛选状态
2. 页面不散写业务口径
3. 字段名和接口文档保持一致
4. 库存类指标逻辑优先写清楚，不做隐式判断
5. 先保证 Summary 模块稳定，再扩展后续模块
