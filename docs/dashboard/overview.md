# Dashboard 产品概览

## 1. 文档信息

- 模块名称：首页 Dashboard
- 版本：v1.0
- 状态：可开发
- 适用门店：咸阳市沣西吾悦广场 3 楼 小黑托昵
- 适用品类：儿童棉品、家居服
- 使用角色：老板、店员
- 前端：Vben v5
- 后端：FastAPI
- 存储：MySQL-ready

---

## 2. 业务背景

当前 POS 系统服务于单店儿童棉品 / 家居服零售门店，具有以下特征：

- 单店经营，没有多店协同需求
- 商品存在明显的尺码、颜色、款式变体
- 当前库存压力较大
- 商品存在明显季节性
- 使用人数较少，主要由老板和店员使用
- 首页更需要承担“经营驾驶舱 + 库存风险提醒 + 明细入口”的作用

本项目当前阶段不适合做重型 BI，也不适合优先做复杂财务分析，更适合做轻量、稳定、可执行的零售首页。

---

## 3. 页面目标

### 3.1 目标

本页面的主要目标：

1. 快速了解当前销售表现
2. 快速识别库存风险
3. 快速发现缺码和过季问题
4. 为补货、清货、商品调整提供入口
5. 为后续分析页提供导航入口

### 3.2 非目标

当前版本不承担以下职责：

1. 不做完整财务报表
2. 不做复杂毛利实时分析
3. 不做多店横向对比
4. 不做复杂会员 CRM 分析
5. 不替代详细销售/库存分析页

---

## 4. 用户角色

### 4.1 老板 / 店主

关注点：

- 今天卖得怎么样
- 订单和客单有没有变化
- 哪些货有库存风险
- 哪些款已经缺码影响销售
- 是否存在过季库存积压

### 4.2 店员

关注点：

- 当前销售状态
- 当前库存预警
- 当前缺码款
- 哪些货需要优先处理

---

## 5. 页面结构

Dashboard 首页 v1 建议由以下模块组成：

1. 顶部 8 张 Summary 卡
2. 销售趋势模块
3. 热销商品 / 热销尺码模块
4. 库存风险模块
5. 会员概览模块

### 5.1 顶部 8 张 Summary 卡

用途：

- 展示经营核心结果
- 展示门店库存风险概况
- 提供进入明细页的快捷入口

### 5.2 销售趋势模块

用途：

- 辅助判断近 7 天 / 30 天趋势变化
- 解释顶部销售指标变化

### 5.3 热销商品 / 热销尺码模块

用途：

- 解释销售件数、客单价、连带率
- 查看哪些商品和尺码在推动销售

### 5.4 库存风险模块

用途：

- 承接顶部库存类指标
- 展示低库存、缺码、过季库存明细

### 5.5 会员概览模块

用途：

- 展示会员销售占比、会员订单占比、复购会员等
- 会员模块不放入顶部 8 卡，但建议保留在首页下半部分

---

## 6. 顶部 8 张卡最终结论

顶部 8 张卡固定为：

1. 销售额 `salesAmount`
2. 订单数 `orderCount`
3. 客单价 `avgOrderValue`
4. 销售件数 `salesQuantity`
5. 连带率 `attachRate`
6. 低库存 SKU 数 `lowStockSkuCount`
7. 缺码款数 `sizeBreakStyleCount`
8. 过季库存件数 `outOfSeasonStockQty`

### 6.1 指标分层

#### 经营结果层

- 销售额
- 订单数
- 客单价
- 销售件数

#### 销售效率层

- 连带率

#### 库存风险层

- 低库存 SKU 数
- 缺码款数
- 过季库存件数

### 6.2 排列建议

#### 第一排：经营结果

- 销售额
- 订单数
- 客单价
- 销售件数

#### 第二排：效率与库存风险

- 连带率
- 低库存 SKU 数
- 缺码款数
- 过季库存件数

---

## 7. 页面设计原则

1. 首页优先展示结果、效率与风险，不堆过多图表
2. 指标口径统一，避免页面与接口语义不一致
3. 页面所有模块共用统一日期筛选状态
4. 销售类指标和库存类指标的联动逻辑必须区分
5. 首页是经营驾驶舱，不是详细报表页
6. 库存风险优先，贴近当前门店实际问题

---

## 8. v1 范围

### 8.1 本期范围

- 顶部 8 张卡
- 日期筛选与联动
- 销售趋势模块
- 库存风险模块
- 热销商品模块
- Summary API 与 Swagger 文档

### 8.2 本期不做

- 毛利实时计算
- 多店切换
- 高级会员分析
- 自动季节切换策略
- 主销码复杂权重规则
- 过季库存金额分析

---

## 9. 跳转关系

- 销售额 → 销售明细页
- 订单数 → 订单列表页
- 客单价 → 销售分析页
- 销售件数 → 商品销售分析页
- 连带率 → 商品组合分析页
- 低库存 SKU 数 → 库存预警页
- 缺码款数 → 缺码明细页
- 过季库存件数 → 过季库存页
