# 平台总体介绍

状态：`Working doc`

## 1. 平台定位

Black Tonny 是一套面向零售经营分析与运营协同的平台。

它的目标不是只做单一报表，而是把零售经营中的：

- 数据观察
- 页面分析
- 指标解释
- 经营协同
- 后续优化动作

逐步连接成一套更稳定、可持续演进的工作系统。

当前平台以服饰零售场景为主，重点关注门店经营、销售与库存观察、经营分析页面和后续数据底座建设。

## 2. 适合谁

当前更适合以下对象：

- 服饰零售品牌主理人
- 单店或多店经营负责人
- 需要逐步建立数据化经营能力的零售团队
- 希望把 ERP 数据沉淀成长期经营资产的业务团队

## 3. 核心价值

### 3.1 让经营分析页面更贴近业务动作

平台不是只展示数字，而是希望把分析页面和实际经营动作连接起来，让管理者更容易理解“发生了什么”和“下一步该做什么”。

### 3.2 为后续稳定数据底座做前置准备

当前前后端协同建设的重点之一，是把分散的页面、接口和业务指标逐步沉淀成可验证、可演进的数据能力。

### 3.3 在建设过程中保留边界感

平台不会把还不稳定的数据能力包装成已完成，而是明确区分哪些已经具备，哪些还在建设中，哪些只是规划方向。

## 4. 当前核心模块

### 4.1 经营分析页面

- dashboard 主页面
- 明细、月度、季度、关系等业务页面
- 指标卡片、结构分析、联动展示和辅助信息区块

### 4.2 数据与服务能力

- summary API 对接
- 页面分析所需的服务层输出
- 数据研究、对账和准入机制

### 4.3 协作与扩展能力

- 前后端拆分协作
- 跨仓库开发协作方式
- 后续更多业务域的扩展基础

## 5. 当前能力边界

### 已具备

- 管理分析前端页面骨架和主交互壳
- dashboard 主路径及其核心页面入口
- 前后端协同的页面与数据接口基础
- 当前账号可见范围内的主要 ERP 路线研究底座
- 面向后续数据准入的状态板与路线注册表

### 已在建设中

- 销售、库存等高价值数据路线的正式准入
- 更稳定的原始数据导入与服务层投影
- 逐步减少 sample/fallback 依赖
- 更完整的分析与经营动作支撑

### 规划中

- 更多业务域的数据准入
- 更成熟的经营建议闭环
- 更完整的客户交付与部署方式
- 更广的组织、角色和门店范围扩展

## 6. 典型使用场景

- 看经营概览和关键指标变化
- 观察销售与库存结构
- 结合分析页面理解当前业务问题
- 作为后续补货、清货和结构优化的讨论基础
- 在客户沟通中说明系统当前能力和建设路径

## 7. 当前不应该怎么理解

当前平台还不应被理解为：

- 已完全替代 ERP 的全量业务系统
- 所有接口和数据路线都已经稳定接入
- 所有经营指标都已经完成最终自动化闭环

更准确的理解是：

平台已经搭起了页面层、服务层和数据研究层的协作骨架，并正在把高价值路线逐步从“能跑”推进到“能信”。
